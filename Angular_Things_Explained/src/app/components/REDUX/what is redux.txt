Redux е degin pattern който държи всички данни на цялото приложение в един единствен JavaScrip object
Това ни дава предвидим глобален state по всяко едно време
В React се казва Redux, а в Angular този патерн се прилага чрез библиотеката Ngrx, която използва Redux принципите с rxJs библиотеката

Принципите в Redux са:
1. Имаме UI който ни показва някакви данни
2. User-а ивзършва някакво действие, което ще променя State на данните - В Redux единствения начин да променим State е чрез dispatch a action
3. Веднъж dispatched action отива в Reducer function, който копира текущото състояние на приложението и промените в нов JavaScript object
Важно е да се отбележи, че State-а е immutable т.е. винаги се прави нов JS object, а не се презаписва стария - така може да се прови проследяване
4. След като Reducer-а генерира новия Store (JS object) може да го запише в client side DB. В ngRx третираме data store като Observable и 
следователно можем да се subscre-нем (абонираме) за него навсякъде в приложението - това означава, че всички наши компоненти и сървиси ще споделят
едни и същи данни в точно даден момент
